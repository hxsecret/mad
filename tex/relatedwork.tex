\section{Related Work}

\noindent \textbf{Adversarial Attacks and Defenses on AI Models.}
Prior AI security research has focused on threats that manipulate inference outcomes or training integrity, including adversarial attacks~\cite{10510296, Ren_2025_CVPR, Xie_2025_CVPR} (imperceptible perturbations causing misclassification), data poisoning~\cite{Bowen_Murphy_Cai_Khachaturov_Gleave_Pelrine_2025, 287222, 10461694} (corrupting training data), and backdoor attacks~\cite{3616617, 10646729,11023440} (injecting hidden triggers). Defenses such as adversarial training and robust distillation address the statistical vulnerabilities of ML algorithms. In contrast, our work targets \textit{AI supply chain security}, where attackers embed malicious code into model artifacts to execute system-level attacks on the victim's host, rather than manipulating model predictions.

\noindent \textbf{Malicious Code Attacks in the AI Supply Chain.}
Attackers embed malicious code into model artifacts on open-source hubs. Early work targets insecure serialization: Python's \texttt{pickle} can execute arbitrary code during deserialization~\cite{casey2024largescaleexploitinstrumentationstudy}. Static tools such as Fickling~\cite{fickling}, Picklescanning~\cite{picklescanning}, and ModelScan~\cite{modelscan} (integrated with ClamAV~\cite{clamav}) detect malware signatures by decompiling bytecodes or matching patterns. However, these signature-based methods are susceptible to evasion and produce high false positives when legitimate operations resemble malicious patterns.

Recent attacks have evolved to framework-native methodologies that abuse built-in APIs for stealthy execution. Zhu et al.~\cite{zhu2025} introduced TensorAbuse, embedding persistent attacks in the model's computational graph via TensorFlow APIs. Using officially supported operations, these attacks bypass traditional static scanners. While the authors proposed an LLM-based tool to statically classify API capabilities, it struggles with semantic ambiguity in distinguishing benign from malicious intents. In contrast, MAD employs dynamic, cross-layer runtime monitoring with high-level execution context to accurately identify malicious intent, overcoming the limitations of static and signature-based defenses.

\noindent \textbf{System Call-Based Detection.}
System call analysis is a core malware detection technique, yet remains underexplored for AI model detection.\cite{}
Recent studies employ deep learning to capture complex API behaviors.\cite{102526,101007} API2Vec++~\cite{10586831} uses temporal and attribute graph embeddings to characterize contextual behaviors across processes, addressing interleaved API calls. API-MalDetect~\cite{103704} applies CNNs and BiGRUs to API sequences for deep semantic feature extraction. CruParamer~\cite{9715123} incorporates parameter sensitivity into API embeddings. Inspired by these advances, MAD adapts system-call monitoring to AI scenarios, providing a framework-agnostic detection foundation that resists application-layer obfuscation.

\noindent \textbf{Unsupervised Anomaly Detection.}
Unsupervised learning excels at zero-day anomaly detection without labeled attack data~\cite{J230570}. Reconstruction-based models like AutoEncoders establish benign behavioral baselines in security applications~\cite{s10462,mirsky2018kitsuneensembleautoencodersonline}. ContraMTD~\cite{3645479} applied ensemble AutoEncoders to anomalous network traffic; similar techniques detect hardware trojans~\cite{10405958} and vision anomalies~\cite{Zhang_2023_ICCV}. MAD employs a sequence-to-sequence AutoEncoder to model the regular execution templates of AI frameworks, flagging structural deviations from stealthy supply-chain attacks without prior malware signatures.
